access-hood

View on GitHub

Zero-dependency, client-side access hood for React. It provides a lightweight visual gate that asks for a password once, verifies it against your backend (when configured), and stores an obfuscated flag in localStorage to reveal gated content on subsequent visits.

Installation

Install in your project:

npm install access-hood

Quick start

Configure remote verification and wrap protected content with AccessHood. Users enter a shared password once per browser: 

import React from "react";
import { AccessHood, setConfig } from "access-hood";

// Configure once at app startup (e.g., in your root file)
setConfig({
  verifyUrl: "/api/ah-verify", // can be absolute or relative; required
  // requestTimeoutMs: 8000,   // optional, defaults to 8000
  passwordHint: "Ask the team lead",
  metadata: {
    title: "Protected Preview",
    description: "This is a protected preview",
  },
  theme: {
    buttonBackground: "#2563eb",
    buttonBorder: "#1e3a8a",
    buttonText: "#ffffff",
    containerBorder: "#e2e8f0",
    hintText: "#475569",
    errorText: "#dc2626",
  },
});

export const Demo = () => {
  return (
    <AccessHood>
      <main>
        <h1>Private content</h1>
        <p>Visible after entering the password once.</p>
      </main>
    </AccessHood>
  );
};

Features

Client-side visual gate

Shows a minimal password form until unlocked. When configured with a verifyUrl, it verifies the password against your backend before unlocking. 

<AccessHood>{content}</AccessHood>

One-time unlock per browser

On successful verification, an obfuscated key/value derived from a configured storage identifier is stored in localStorage, so the content stays unlocked for subsequent visits.

Theming

Override colors via the theme option in setConfig or compute styles with getStyles for reuse. 

import { getStyles } from "access-hood";
const styles = getStyles({ buttonBackground: "#2563eb" });

Metadata

Set document.title when the gate mounts using the optional metadata option in setConfig. 

setConfig({ metadata: { title: "Protected Preview" } });

TypeScript-first and zero-dependency

Ships types and no runtime dependencies. Uses Web Crypto when available with a tiny deterministic fallback.

Notes and limitations

  • Client-side gate only; even with a remote check, do not ship sensitive data unless authorized server-side.
  • SSR: The gate renders on the client. Initial server render returns nothing; the form appears after hydration.
  • Changing the password or clearing site data resets the gate.

API Reference

Exports

  • AccessHood — React component that renders the gate and your children when unlocked
  • setConfig — configure remote verification
  • getConfig — read the current remote verification config
  • getStyles — helper to compute the inline styles used by the component
  • AccessHoodTheme — TypeScript type for theme overrides

AccessHood props

  • children: React.ReactNode — Content to show after access is granted
